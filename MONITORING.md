# Monitoring Stack: Loki + Umami

Plan to add centralised log aggregation (Loki) and web analytics (Umami) to the home lab.

## Context

- **Infrastructure**: Proxmox VE on `pve-node-1` (192.168.1.85), services run as LXC containers
- **Provisioning method**: [tteck Proxmox community scripts](https://tteck.github.io/Proxmox/) — each service gets a dedicated LXC container
- **Loki**: Aggregates logs from Docker containers running inside existing LXC hosts
- **Umami**: Web analytics for the Next.js app in LXC container 107

## Missing context / prerequisites

Before the playbooks can fully run, the following must be known or confirmed:
- IPs assigned to new `loki` and `umami` LXC containers (DHCP — only known after provisioning)
- Which existing LXC hosts run Docker (populate `[docker-hosts]` in inventory)
- Umami `APP_SECRET` and database credentials (generated by tteck script; rotate post-install)
- Next.js app structure in LXC 107 (to embed the Umami tracking script)

## Playbook Designs

### `monitoring_provision.yml` — create LXC containers

Runs on `pvenodes`. Downloads and executes tteck community scripts non-interactively.

> **Note**: tteck scripts are not idempotent — run once only.

```yaml
- hosts: pvenodes
  tasks:

  - name: install curl
    apt:
      name: curl
      update_cache: yes
      cache_valid_time: 3600
      state: latest

  - name: create Loki LXC using tteck community script
    shell: >
      bash -c "$(curl -fsSL https://raw.githubusercontent.com/tteck/Proxmox/main/ct/loki.sh)"
    environment:
      VERBOSE: "yes"
      INT_OR_EXT: "INT"
      CT_TYPE: "1"
      DISK_SIZE: "8"
      CORE_COUNT: "2"
      RAM_SIZE: "512"
      NET_TYPE: "dhcp"
      HN: "loki"
    args:
      executable: /bin/bash

  - name: create Umami LXC using tteck community script
    shell: >
      bash -c "$(curl -fsSL https://raw.githubusercontent.com/tteck/Proxmox/main/ct/umami.sh)"
    environment:
      VERBOSE: "yes"
      INT_OR_EXT: "INT"
      CT_TYPE: "1"
      DISK_SIZE: "4"
      CORE_COUNT: "1"
      RAM_SIZE: "512"
      NET_TYPE: "dhcp"
      HN: "umami"
    args:
      executable: /bin/bash
```

After running, find the assigned IPs:
```bash
pvesh get /nodes/pve-node-1/lxc --output-format json | grep ip
```
Update `inventory` TBD values, then onboard:
```bash
ansible-playbook pve_onboard.yml --limit loki,umami
```

### `monitoring_configure_loki.yml` — configure Loki

Targets `monitoring-loki`. Deploys `/etc/loki/loki-local-config.yaml`:

- `auth_enabled: false`
- HTTP listen port `3100`
- Schema `v13` with `tsdb` index (current recommended, avoids future migration)
- Filesystem storage at `/var/lib/loki/{chunks,rules,compactor}`
- `retention_period: 720h` (30 days) via compactor
- `analytics.reporting_enabled: false`

```yaml
- hosts: monitoring-loki
  tasks:

  - name: ensure loki config directory exists
    file:
      path: /etc/loki
      state: directory
      owner: root
      group: root
      mode: '0755'

  - name: deploy loki configuration
    copy:
      dest: /etc/loki/loki-local-config.yaml
      owner: root
      group: root
      mode: '0644'
      content: |
        auth_enabled: false

        server:
          http_listen_port: 3100
          grpc_listen_port: 9096

        common:
          path_prefix: /var/lib/loki
          storage:
            filesystem:
              chunks_directory: /var/lib/loki/chunks
              rules_directory: /var/lib/loki/rules
          replication_factor: 1
          ring:
            instance_addr: 127.0.0.1
            kvstore:
              store: inmemory

        schema_config:
          configs:
            - from: 2024-01-01
              store: tsdb
              object_store: filesystem
              schema: v13
              index:
                prefix: index_
                period: 24h

        limits_config:
          retention_period: 720h
          ingestion_rate_mb: 4
          ingestion_burst_size_mb: 6

        compactor:
          working_directory: /var/lib/loki/compactor
          compaction_interval: 10m
          retention_enabled: true
          retention_delete_delay: 2h
          retention_delete_worker_count: 150

        analytics:
          reporting_enabled: false
    notify: restart loki

  - name: ensure loki storage directories exist
    file:
      path: "{{ item }}"
      state: directory
      owner: loki
      group: loki
      mode: '0755'
    loop:
      - /var/lib/loki
      - /var/lib/loki/chunks
      - /var/lib/loki/rules
      - /var/lib/loki/compactor

  - name: ensure loki service is enabled and started
    systemd:
      name: loki
      enabled: yes
      state: started

  handlers:
  - name: restart loki
    systemd:
      name: loki
      state: restarted
```

### `monitoring_configure_docker_logging.yml` — ship Docker logs to Loki

Targets `docker-hosts`. Loki URL is derived dynamically from inventory (no hard-coded IPs).

```yaml
- hosts: docker-hosts
  vars:
    loki_url: "http://{{ hostvars['loki']['ansible_host'] }}:3100/loki/api/v1/push"
  tasks:

  - name: install docker
    apt:
      name: docker.io
      update_cache: yes
      cache_valid_time: 3600
      state: latest

  - name: ensure docker service is enabled and started
    systemd:
      name: docker
      enabled: yes
      state: started

  - name: install loki docker logging driver plugin
    shell: >
      docker plugin install grafana/loki-docker-driver:latest
      --alias loki
      --grant-all-permissions
    args:
      executable: /bin/bash
    register: plugin_install
    failed_when:
      - plugin_install.rc != 0
      - '"already installed" not in plugin_install.stderr'
    changed_when: '"already installed" not in plugin_install.stderr'

  - name: configure docker daemon to use loki as default log driver
    copy:
      dest: /etc/docker/daemon.json
      owner: root
      group: root
      mode: '0644'
      content: |
        {
          "log-driver": "loki",
          "log-opts": {
            "loki-url": "{{ loki_url }}",
            "loki-retries": "5",
            "loki-batch-size": "400",
            "loki-external-labels": "container_name={{.Name}},host={{ inventory_hostname }}"
          }
        }
    notify: restart docker

  handlers:
  - name: restart docker
    systemd:
      name: docker
      state: restarted
```

> **Warning**: Docker daemon restart briefly stops all containers on that host.
> The `{{.Name}}` in `loki-external-labels` is a Docker template (not Ansible) — the double braces are intentional.

### `monitoring_configure_umami.yml` — configure Umami

Targets `monitoring-umami`. Ensures the service is running and accessible.

```yaml
- hosts: monitoring-umami
  tasks:

  - name: ensure umami service is enabled and started
    systemd:
      name: umami
      enabled: yes
      state: started

  - name: check umami is reachable on port 3000
    wait_for:
      host: 127.0.0.1
      port: 3000
      timeout: 30
      state: started

  - name: post-install reminders
    debug:
      msg: >
        Umami is running at http://{{ ansible_host }}:3000
        Default credentials: admin / umami — change immediately.
        Rotate APP_SECRET in the umami environment file
        (check: systemctl cat umami | grep EnvironmentFile).
```

#### Next.js integration in LXC 107 (manual steps)

1. Log in to Umami at `http://<umami-ip>:3000`
2. Settings → Websites → Add website → copy the generated `data-website-id` UUID
3. In the Next.js app (`/app/layout.tsx`), add:
   ```tsx
   import Script from 'next/script'

   // inside <head> or at end of <body>:
   <Script
     defer
     src="http://<umami-ip>:3000/script.js"
     data-website-id="<your-uuid>"
     strategy="afterInteractive"
   />
   ```
4. Redeploy the Next.js app in LXC 107
